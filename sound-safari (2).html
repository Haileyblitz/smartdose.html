🔬 Developer Note — AI Concepts Used in This App

This app demonstrates two foundational concepts from symbolic / rule-based AI:

1. Knowledge Representation

All game content lives in a structured knowledge base — a JavaScript object called KNOWLEDGE_BASE. Each entry maps a phoneme target (like "S") to a list of items that begin with that sound, along with their emoji and labels. This mirrors how expert systems represent domain facts: each "fact" is an explicit, structured relationship between a concept and its properties. The machine doesn't understand language — it stores and retrieves pre-defined associations.

2. Pattern Matching

When a child taps a card, the app runs a pattern matching routine. Each click is evaluated against a rule: "Does this item's key appear in the correct set for the active target?" This is a classic IF–THEN production rule — the same architecture used in early AI expert systems like MYCIN. The "intelligence" is the lookup: correctKeys.includes(clickedKey).

3. Constrained Randomness (Distractor Generation)

The grid is populated with a controlled shuffle. Correct items are always guaranteed to appear; distractors are drawn randomly from the other phoneme lists. This is constraint-based generation — randomness operating within logical rules — similar to how AI puzzle generators ensure solvability while maintaining variety each round.

Together, these techniques show that AI isn't always neural networks — structured knowledge + logical rules can create meaningful, adaptive learning experiences.
